Pourquoi une refonte ?

  • Application devenue trop hétérogène au fil des années
  • Difficulté pour maintenir l'application
  • Impossibilité de tester l'application front


React

  • Créée en 2013 par Facebook
  • Plus une librairie qu'un framework
  • Souvent considérée comme le V de MVC
  • Isomorphisme

Comment gérer une UI amenée à se transformer ?

Data-binding

  • Rendu de l’UI à l’initialisation
  • Mise à jour des "bindings" quand les données changent

Et si on réinitialisait la vue à chaque changement ?

La solution React :
le "Virtual DOM"

Le DOM n'est pas optimisé pour gérer des UI dynamiques.
Soit, utilisons une abstraction de ce DOM, un DOM virtuel.

Et concrètement ?

Dans React tout est composant, et chaque composant possède une méthode render.

  1. render est appelée à l'initialisation du composant
  2. Le Virtual DOM puis le DOM sont mis à jour
  3. render est à nouveau appelée quand les données du composant changent
  4. Le Virtual DOM est mis à jour
  5. React compare l'ancienne représentation du composant avec la nouvelle : "Reconciliation"
  6. Le DOM est mis à jour
